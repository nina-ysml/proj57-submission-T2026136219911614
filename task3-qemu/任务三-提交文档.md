# 任务三：StarryOS / QEMU 上 ACT 模型推理部署文档

## 一、完成概况

在 StarryOS（RISC-V 64，QEMU 虚拟机）上成功完成 ACT 模型的 CPU 推理。使用 ONNX Runtime C API 加载模型并执行前向计算，输出预测的动作序列。

## 三、关键数据

| 项目 | 数值 |
|------|------|
| 推理程序大小 | 13 KB |
| ONNX 模型大小 | 193 MB |
| 运行时依赖库 | 36 MB（libonnxruntime 19M + libstdc++ 17M + libatomic 140K + libgcc 584K） |
| 推理时长 | 约 215 秒（QEMU RISC-V 模拟） |
| 推理运行时内存 | 约 230 MB（模型 193M + ONNX Runtime 会话 ~30M + 输入/输出缓冲 ~7M） |

| 输出 shape | [1, 8, 3]（batch × chunk_size × action_dim） |

> 注：StarryOS 不提供进程级内存接口（free 命令不可用、/proc/self/status 不存在），运行时内存为模型加载量与 ONNX Runtime 开销的估算值。

## 六、部署流程

### 1. 导出 ONNX 模型

使用 ACTExportWrapper 包装器，`infer_cvae=False` 跳过 CVAE 随机采样，导出 5D 输入格式的 ONNX 模型。

### 2. 预处理输入数据

- 图片：JPEG → resize 224×224 → float32 → ImageNet 归一化 → [1, 1, 3, 224, 224]
- 状态：[0.0, 0.0] → Q01/Q99 归一化 → [-0.434, -1.0]
- 反归一化参数：从 stats.json 提取 action Q01/Q99

### 3. 交叉编译推理程序

```bash
riscv64-linux-musl-gcc -O2 -I<ort_headers> act_onnx_c.c \
  -L<runtime_libs> -lonnxruntime -Wl,-rpath,/lib -lm -o act_onnx_c_riscv64
```

### 4. 编译 StarryOS

```bash
git clone https://github.com/Starry-OS/StarryOS.git
cd StarryOS
make ARCH=riscv64 rootfs
make ARCH=riscv64 build
```

### 5. 注入文件到 rootfs

使用 debugfs 将推理程序、模型、运行时库、输入数据注入 StarryOS 根文件系统镜像。

### 6. 启动 QEMU 并运行推理

```bash
qemu-system-riscv64 -machine virt -m 1G -nographic -cpu rv64 -smp 1 \
  -kernel starryos.bin \
  -device virtio-blk-pci,drive=disk0 \
  -drive id=disk0,format=raw,file=rootfs.img
# 进入 shell：/root/act/act /root/act
```
